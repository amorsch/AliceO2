113  size_t pushBack(const char* ptr, size_t sz, bool keepLastOnFlash = true)
114  {
115  if (!sz) {
116  return buffer.size();
117  }
118  nBytesWritten += sz;

121  flushSuperPage(keepLastOnFlash);

123  auto offs = expandBufferBy(sz);

203  // This is a placeholder for the function responsible to split large payload to pieces
204  // fitting 8kB CRU pages.
205  // The RawFileWriter receives from the encoder the payload to format according to the CRU format
206  // In case this payload size does not fit into the CRU page (this may happen even if it is
207  // less than 8KB, since it might be added to already partially populated CRU page of the HBF)
208  // it will write on the page only part of the payload and carry over the rest on extra page(s).
209  // By default the RawFileWriter will simply chunk payload as is considers necessary, but some
210  // detectors want their CRU pages to be self-consistent and in case of payload splitting they
211  // add in the end of page to be closed and beginning of the new page to be opened
212  // (right after the RDH) detector-specific trailer and header respectively.
213  //
214  // The role of this method is to suggest to writer how to split the payload:
215  // If this method was set to the RawFileWriter using
216  // RawFileWriter::setCarryOverCallBack(pointer_on_the_converter_class);
217  // then the RawFileWriter will call it before splitting.
218  //
219  // It provides to the carryOverMethod method the following info:
220  // rdh : RDH of the CRU page being written
221  // data : original payload received by the RawFileWriter
222  // ptr : pointer on the data in the payload which was not yet added to the link CRU pages
223  // maxSize : maximum size (multiple of 16 bytes) of the bloc starting at ptr which it can
224  // accomodate at the current CRU page (i.e. what it would write by default)
225  // splitID : number of times this payload was already split, 0 at 1st call
226  // trailer : what it wants to add in the end of the CRU page where the data starting from ptr
227  // will be added. The trailer is supplied as an empy vector, which carryOverMethod
228  // may populate, but its size must be multiple of 16 bytes.
229  // header : what it wants to add right after the RDH of the new CRU page before the rest of
230  // the payload (starting at ptr+actualSize) will be written
231  //
232  // The method mast return actual size of the bloc which can be written (<=maxSize).
233  // If this method populates the trailer, it must ensure that it returns the actual size such that
234  // actualSize + trailer.size() <= maxSize
235  // In case returned actualSize == 0, current CRU page will be closed w/o adding anything, and new
236  // query of this method will be done on the new CRU page
237 
238  int carryOverMethod(const RDH& rdh, const gsl::span<char> data, const char* ptr, int maxSize, int splitID,
239  std::vector<char>& trailer, std::vector<char>& header) const

241  return maxSize; // do nothing
